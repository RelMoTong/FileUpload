# Version 3.2.0 - 文件清理与自动清理交互升级

## 发布日期
2026-04-13

## 本次版本重点

### 1. 文件清理界面完成交互升级
- 结果区从简单日志升级为交互式表格。
- 支持复选框批量选择、列排序、关键字搜索和右键菜单操作。
- 扫描阶段支持取消，并显示当前目录、文件数量和累计大小。
- 删除前会展示摘要信息，降低误删风险。

### 2. 自动清理逻辑补齐到完整配置链路
- 自动清理现在支持保留天数、格式过滤和回收站删除模式。
- 配置可以从界面保存到 `config.json`，并在主窗口重新加载回填。
- Worker 只负责发现磁盘不足并发出信号，真正的清理由主窗口统一执行。
- 紧急清理模式会透传到统一清理任务，避免告警和执行逻辑脱节。

### 3. 权限与按钮联动进一步收敛
- `guest` 角色不能执行磁盘清理和自动清理配置保存。
- 上传运行中禁止修改上传配置和清理配置。
- 删除按钮只会在“当前有勾选文件且当前角色有权限”时启用。
- 修改密码入口收敛为仅 `admin` 可用。

### 4. 启动和配置交互细节修正
- 配置加载期间增加守卫，避免启动回填时误标记“配置已修改”。
- 上传前配置日志中的过滤条件定义顺序修正，避免旧版本中的未定义问题。
- 自动清理配置界面与主窗口实际执行参数保持一致，不再出现“界面看不到、逻辑却在用”的隐性配置。

### 5. 打包与发布链路同步整理
- 版本号统一提升到 `3.2.0`，并继续以 `src.__version__` 作为单一来源。
- PyInstaller 构建固定使用 `PySide6`，避免环境中同时存在 `PyQt5` 时打包失败。
- 增加 `Send2Trash` 依赖，确保打包版保留回收站删除能力。
- 发布产物与文档版本保持一致，便于 GitHub 发布和追踪。

## 适合 GitHub 发布页的摘要
- 文件清理 UI 升级为可交互表格，支持搜索、排序、右键菜单和删除前摘要。
- 自动清理补齐保留天数、格式过滤、回收站模式，并统一到主窗口调度。
- 权限和按钮联动修正，运行中与访客态行为更加一致。
- 打包版本号、发布文档和依赖项同步整理。
