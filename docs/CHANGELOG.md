# 版本摘要（精简版）

## v3.3.0 (2026-04-29)
- 磁盘空间不足告警增强：SMB 写入遇到目标盘爆满时明确记录 `DISK_FULL`，避免被误判为网络断开或复制超时。
- FTP 上传错误透传增强：保留 FTP 最近一次错误信息，远端空间不足、配额不足等场景会记录 `FTP_DISK_FULL`。
- 自动清理日志增强：删除失败时记录具体文件路径和失败原因，最多保留前 10 条明细，便于现场排查文件占用、权限或回收站异常。
- 自动清理结果告警增强：清理后仍未达到目标阈值时，记录剩余需释放空间，提醒检查权限、文件占用或扩大清理目录。
- 打包版本提升到 `3.3.0`，生成 `ImageUploadTool_v3.3.0` 现场实施包。

## v3.2.1 (2026-04-15)
- 自动清理保存逻辑增加保护：当当前界面未勾选任何可见清理目录时，禁止继续保留历史隐藏目录，避免误以为“未选路径”仍会执行清理。
- 补充回归测试，覆盖自动清理隐藏目录保留场景与触发执行参数传递。

## v3.2.0 (2026-04-13)
- 文件清理界面升级为可勾选表格，支持排序、搜索、右键操作、扫描进度、删除前摘要和更清晰的交互反馈。
- 自动清理配置补齐到界面层，支持保留天数、格式过滤、回收站删除模式，并与主窗口配置保存/回填链路统一。
- 自动清理执行链路调整为由主窗口统一调度，Worker 仅负责告警；紧急清理模式会透传到统一清理任务。
- 权限与按钮联动收敛：`guest` 不能执行清理和配置保存，运行中禁止修改配置，删除按钮只跟随勾选文件启用。
- 配置加载守卫补齐，避免启动时把配置回填误记为“用户手动修改”。
- 打包链路对齐版本号，PyInstaller 固定使用 `PySide6`，并补充 `Send2Trash` 依赖，保证回收站功能在打包版中可用。
- 详细说明见 `RELEASE_NOTES_v3.2.0.md`、`docs/GITHUB_RELEASE_v3.2.0.md` 和 `docs/CLEANUP_UPGRADE.md`。

## v3.1.1
