# Vision PLC Document Writer Skill

## Purpose

Use this skill for industrial machine vision and PLC-related writing.

Typical tasks:

- English emails to customers or vendors
- Chinese-English technical explanations
- inspection logic documentation
- PLC handshake explanation
- retest instructions
- debugging reports
- Keyence / Cognex / camera system instructions
- on-site change notes

## Writing Rules

1. Be clear, professional, and technically precise.
2. Avoid overly casual wording in customer-facing content.
3. Do not sound submissive or uncertain unless there is real uncertainty.
4. Separate facts, assumptions, risks, and recommendations.
5. Use bilingual Chinese-English output when useful.
6. For emails, keep the tone polite and direct.
7. For manuals, use numbered steps.

## Technical Structure

When explaining a vision inspection process, include relevant items:

- station name
- trigger condition
- image acquisition timing
- inspection region
- feature used for positioning
- OK / NG judgment
- PLC signal interaction
- result confirmation
- retest method
- image/history saving
- known risks
- recommended next step

## Output Formats

For email:

- Subject
- English version
- Optional Chinese explanation

For manuals:

- Title
- Preconditions
- Steps
- Notes
- Common problems

For technical proposals:

- Current issue
- Cause analysis
- Proposed method
- Advantages
- Risks
- Verification method
