# UI Refactor Ant Design Pro Skill

## Purpose

Use this skill when the user asks to improve a management system UI, page layout, table layout, forms, filters, modals, responsive behavior, or visual hierarchy.

## Main Rule

Do not change business logic unless the user explicitly requests it.

## Style Direction

Use a modern enterprise management system style inspired by Ant Design Pro:

- clean page structure
- clear information hierarchy
- unified spacing
- unified table style
- unified forms
- unified modals
- practical responsive behavior
- no flashy gradients
- no decorative animation
- no unnecessary visual tricks

## Refactor Priorities

1. Page structure
2. Navigation and module grouping
3. Filter/search area
4. Table readability
5. Form layout
6. Modal layout
7. Button priority
8. Empty/loading/error states
9. Responsive behavior
10. Visual consistency

## Output Requirements

Before editing:

- list current UI problems
- explain the target layout
- provide a refactor plan

After editing:

- list changed files
- summarize layout changes
- explain which business logic was preserved
- provide test steps
